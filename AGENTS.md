# OpenAI Build Week project guidance

This repository is being built for OpenAI Build Week (the Codex/GPT-5.6 hackathon). Before planning, implementing, reviewing, or preparing the submission, read `HACKATHON_BRIEF.md` and treat it as a product constraint.

Optimize the project and demo for the four equally weighted judging criteria:

1. Technological Implementation
2. Design
3. Potential Impact
4. Quality of the Idea

Keep the product runnable, coherent, non-trivial, easy for judges to test, and clearly differentiated. Preserve evidence of work completed during the submission period and document where Codex accelerated the workflow, which important decisions remained human-led, and how GPT-5.6 contributed.

Do not let feature work jeopardize the hard submission requirements or the deadline. Re-check live Devpost rules and announcements before making eligibility, deadline, or submission claims because the official rules can change.
